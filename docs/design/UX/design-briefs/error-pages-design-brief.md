# Error Pages

---

## Overview

The Error Pages module defines how Pharmora communicates unexpected situations while maintaining a professional and trustworthy user experience.

Instead of displaying generic browser errors or blank screens, Pharmora should provide meaningful explanations, reassure users, and offer clear next actions.

Every error page should remain consistent with the Pharmora Design System and preserve user confidence.

---

# Objective

Help users understand:

- What happened
- Why the page cannot be displayed
- What they can do next

The experience should feel calm, helpful, and informative instead of alarming.

---

# Design Principles

Every error page should:

- Be simple and easy to understand
- Maintain generous whitespace
- Use minimal illustrations
- Feel professional and reassuring
- Provide a clear recovery action
- Avoid technical jargon whenever possible

---

# Layout

All error pages follow the same structure.

Content should be vertically and horizontally centered.

Structure:

1. Illustration
2. Error Code
3. Title
4. Description
5. Primary Button
6. Secondary Button (Optional)

---

# 403 Forbidden

## Trigger

The user attempts to access a page without sufficient permissions.

## Error Code

403

## Title

Access Denied

## Description

You don't have permission to access this page.

If you believe this is a mistake, please contact your administrator.

## Primary Button

Go to Dashboard

## Secondary Button

Contact Support

## Illustration

Shield

Lock

Security

Permission

# 404 Not Found

## Trigger

## Error Code

404

## Title

## Description

The page you're looking for doesn't exist or may have been moved.

Let's help you get back on track.

## Secondary Button

Go Back

# Empty States

---

## Overview

The Empty States module defines how Pharmora communicates with users when no data is available.

Instead of displaying blank tables or empty pages, Pharmora should provide meaningful guidance, explain why the page is empty, and encourage users to perform the next logical action.

Every empty state should feel helpful, welcoming, and consistent with Pharmora's premium enterprise design language.

---

## Objective

Help users understand:

- Why no data is displayed
- What the current page represents
- What action they should take next

The experience should reduce confusion while encouraging user engagement.

---

## Design Principles

Every empty state should:

- Feel clean and minimal
- Maintain generous whitespace
- Avoid overwhelming illustrations
- Use soft healthcare-inspired colors
- Include a clear primary action
- Feel encouraging rather than alarming

---

## Layout

Every empty state follows the same structure.

Centered vertically and horizontally inside the content area.

Order:

1. Illustration
2. Title
3. Description
4. Primary Action
5. Optional Secondary Action

---

## Components

### Illustration

Minimal flat illustration.

Simple.

Modern.

No colorful cartoons.

No 3D graphics.

Rounded style.

Soft green accent.

Light gray background shapes.

---

### Title

Large.

Bold.

Dark Slate.

Example:

No Products Yet

---

### Description

Explain the current situation.

Maximum 2–3 lines.

Friendly.

Professional.

Example:

You haven't added any products yet.

Start building your healthcare inventory by creating your first product.

---

### Primary CTA

Large primary button.

Pharmora Green.

Examples:

Add Product

Create Category

Add Supplier

Update Inventory

Generate Report

---

### Secondary CTA (Optional)

Ghost button.

Learn More

View Documentation

Import Data

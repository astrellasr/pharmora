# 1. Overview

## Purpose

This document defines the low-fidelity wireframe specification for the Reports Dashboard screen.

Its purpose is to establish layout hierarchy, information organization, navigation flow, and interaction patterns before high-fidelity interface design begins.

This document intentionally excludes branding, colors, typography, shadows, and visual styling.

---

# 2. Wireframe Goals

The Reports wireframe should:

- Present business insights clearly.
- Prioritize analytical information.
- Support efficient report generation.
- Reduce cognitive load.
- Maintain consistency with the Pharmora Design System.
- Support responsive layouts.

# 4. Information Hierarchy

Priority order:

1. Page Header
2. Summary Cards
3. Filter Toolbar
4. Analytics Charts
5. Inventory Insights
6. Recent Reports

The analytical charts should become the primary visual focus.

---

# 5. Screen Sections

## Header

Contains:

- Page Title
- Breadcrumb
- Export Report Button

---

## Sidebar

Contains navigation only.

No report data should appear inside the sidebar.

---

## Summary Cards

Arrange horizontally.

Recommended number:

4 Cards

Examples:

- Inventory Value
- Products Sold
- Stock Movements
- Low Stock Alerts

---

## Filter Toolbar

Display:

- Date Range Picker
- Category Filter
- Supplier Filter
- Refresh Button
- Export Button

Toolbar actions should remain on a single row on desktop.

---

## Analytics Charts

Primary visualization area.

Recommended charts:

- Inventory Trend
- Stock Movement
- Category Distribution
- Inventory Value

Charts should occupy the majority of available space.

---

## Inventory Insights

Display key operational insights.

Examples:

- Highest Stock Products
- Lowest Stock Products
- Fast Moving Products
- Most Active Categories

---

## Recent Reports

Display previously generated reports.

Columns:

- Report Name
- Generated By
- Date
- Status

Maximum visible rows:

5

# 9. User Interaction

Primary Actions

- Generate Report
- Export Report

Secondary Actions

- Filter
- Refresh
- Select Date Range

---

# 10. Empty State

If no report data exists:

Display:

- Empty Illustration
- "No Reports Available"
- Supporting Description
- Generate Report Button

---

# 11. Loading State

Display:

- Skeleton Cards
- Skeleton Charts
- Skeleton Tables

Prevent layout shifting during loading.

---

# 12. Error State

Display:

- Error Illustration
- Error Description
- Retry Button

---

# 13. Accessibility

The wireframe should support:

- Keyboard Navigation
- Logical Tab Order
- Accessible Charts
- Consistent Focus Order
- Readable Layout Structure

---

# 14. Design Constraints

The wireframe must:

- Remain grayscale.
- Exclude branding.
- Exclude typography styling.
- Exclude colors.
- Exclude shadows.
- Exclude gradients.
- Focus only on layout and hierarchy.
